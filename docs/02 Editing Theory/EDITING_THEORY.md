# 1. Purpose

Este documento define como a EVA pensa.

Não descreve código.

Não descreve IA.

Descreve a lógica editorial utilizada para tomar decisões.

---

# 2. Fundamental Principle

Editar não significa cortar.

Editar significa tomar decisões.

Todo corte é consequência de uma decisão.

Nunca o contrário.

---

# 3. Objective First

Toda edição começa por um objetivo.

Nunca por um efeito.

Exemplo:

Objetivo

↓

Narrativa

↓

Ritmo

↓

Decisões

↓

Renderização

---

# 4. Editing Goals

Toda edição deve possuir exatamente um objetivo principal.

Exemplos:

- Conversão
- Viralização
- Storytelling
- Educação
- Entretenimento
- Branding
- Tutorial
- Review
- Podcast

Objetivos secundários podem existir.

Mas sempre existe apenas um objetivo dominante.

# 6. Editorial Priorities

A EVA sempre prioriza:

1. Clareza
2. Atenção
3. Narrativa
4. Emoção
5. Estética

Nunca o contrário.

---

# 7. Attention Theory

A atenção do espectador é o recurso mais importante.

Toda decisão deve responder:

"Isso aumenta ou reduz a atenção?"

Se reduzir:

Remover.

Se aumentar:

Manter.

---

# 8. Narrative Theory

Uma boa edição facilita o entendimento.

Toda cena deve responder:

- O que está acontecendo?
- Por que isso importa?
- O que acontece depois?

Se qualquer resposta não existir:

A narrativa precisa ser ajustada.

---

# 9. Information Density

Todo segundo do vídeo deve transmitir informação.

Informação pode ser:

- visual
- emocional
- narrativa
- educativa
- comercial

Tempo sem informação reduz qualidade.

---

# 10. Dead Time

Tempo morto é qualquer intervalo que não aumenta:

- compreensão
- emoção
- atenção
- narrativa

Tempo morto deve ser removido.

---

# 11. Silence

Silêncio não é necessariamente ruim.

Silêncio é removido apenas quando não possui função narrativa.

---

# 12. Emotion

A emoção determina prioridade.

Momentos emocionalmente relevantes recebem prioridade sobre estética.

---

# 13. Visual Hierarchy

A atenção deve ser guiada.

A EVA utiliza:

- enquadramento
- zoom
- contraste
- movimento
- texto
- cor

para direcionar o olhar.

---

# 14. Rhythm

Ritmo é a frequência de mudança de informação.

Mudanças podem ocorrer através de:

- cortes
- zoom
- texto
- movimento
- áudio
- música

---

# 15. Continuity

Mudanças nunca devem quebrar compreensão.

Toda transição deve preservar contexto.

---

# 16. Cognitive Load

A edição nunca deve exigir esforço desnecessário.

Reduzir:

- excesso de texto
- excesso de efeitos
- excesso de cortes
- excesso de informação

---

# 17. Editing Principles

Toda decisão deve melhorar pelo menos um destes fatores:

- compreensão
- retenção
- emoção
- clareza
- impacto

Caso contrário:

Não executar.

---

# 18. Decision Validation

Antes de executar qualquer ação, a EVA pergunta:

Esta decisão possui objetivo?

↓

Possui justificativa?

↓

Melhora o vídeo?

↓

Pode ser explicada?

↓

Pode ser reproduzida?

Somente então a decisão é aprovada.

---

# 19. Explainability

Nenhuma decisão existe sem:

- motivo
- confiança
- agente
- regra
- versão

---

# 20. Reproducibility

A mesma entrada deve produzir o mesmo resultado utilizando:

- mesmas regras
- mesmo conhecimento
- mesma versão

---

# 21. Knowledge Before AI

A IA nunca decide sozinha.

Toda decisão consulta conhecimento.

Conhecimento possui prioridade sobre improvisação.

---

# 22. Learning

Toda edição gera conhecimento.

Conhecimento gera regras.

Regras melhoram futuras decisões.

---

---

# 24. Success Definition

Uma edição é considerada correta quando:

- cumpre o objetivo
- melhora retenção
- melhora compreensão
- reduz esforço cognitivo
- mantém coerência narrativa
- pode ser explicada
- pode ser reproduzida

---

# 25. Golden Rule

A EVA nunca pergunta:

"Qual efeito devo aplicar?"

A EVA pergunta:

"Qual decisão melhora este vídeo?"
